编写通用模块

每个 node 应用都是一个「包」

而「通用模块」就是一种最简单、最基础、最通用的包

主要作用是供其他更为复杂、庞大的应用调用

要开发一个包

需要在应用的根目录建一个 package.json 文件

用来记述这个应用的相关信息

它基本是长成这个样子

{% highlight json %}{ "name": "rocketz-core", "version": "1.0.3", "description": "Upload local files to CDNs", "homepage": "https://github.com/ourai/rocketz", "keywords": ["rocketz", "uploader", "upload", "cdn", "cloud"], "author": "Ourai Lin (http://ourai.ws/)", "repository": { "type": "git", "url": "https://github.com/ourai/rocketz.git" }, "bugs": "https://github.com/ourai/rocketz/issues", "main": "index.js", "scripts": { "prebuild": "mv lib src", "build": "babel src -d lib --presets=es2015", "prepublish": "npm run build", "postpublish": "rm -rf lib && mv src lib" }, "devDependencies": { "babel-cli": "^6.9.0", "babel-preset-es2015": "^6.9.0" }, "dependencies": { "lodash": "^4.1.0", "rocketz-register": "^0.1.1" }, "license": "MIT" }{% endhighlight %}

在入口文件 index.js 中写几行代码

{% highlight js %}// 定义一个 CDN 信息 const CDN_INFO = { qiniu: {name: "七牛"}, wantu: {name: "顽兔"} }; // 暴露出获取指定 CDN 信息的方法 exports.getCdn = function( cdnName ) { return CDN_INFO[cdnName] || {}; };{% endhighlight %}

就完成了一个最「简陋」的包

在其他包中引用一下就可以调用其中定义的方法了

{% highlight js %}// 引入通用模块 const rocketz = require("rocketz-core"); // 调用方法 console.log(rocketz.getCdn("qiniu")); // 输出 {name: "七牛"}{% endhighlight %}

命令行工具

这是一种较为高级的 node 应用

它是通过命令行调用的程序

如 rocketz init

命令行工具与通用模块的不同之处在于

  1. 不可被其他包引用
  2. 需要在 package.json 中指定 "bin"
  3. 针对命令和 flag 的处理逻辑

其中，如果直接写成

{% highlight json %}{ "bin": "./bin/rocketz.js" }{% endhighlight %}

将会以包的名字作为命令名

要想命令名不同于包名或存在多个命令

就要明确写出命令的名字

{% highlight json %}{ "bin": { "rocketz": "./bin/rocketz.js" } }{% endhighlight %}

完全自己去写命令和 flag 的处理逻辑是件让人蛋碎的事情

还好已经有一些第三方包可以起到辅助作用

我主要用到了三个包

  • meow
  • lift
  • inquirer

偷懒万岁！*\(^o^)/*

meow 的主要作用有

  1. 将命令和 flag 分别转化为数组和对象以便于读取
  2. 自动生成 --version 和 --help 这两个 flag
  3. 设置 flag 的别名

meow 的用法很简单

{% highlight js %}const meow = require("meow"); const helper = meow({}, { alias: { h: "help", v: "version" } });{% endhighlight %}

如果在命令行中输入

{% highlight sh %}rocketz init qiniu --config wantu{% endhighlight %}

程序中调用 helper.input 和 helper.flags

{% highlight js %}console.log(helper.input); // 输出 ["init", "qiniu"] console.log(helper.flags); // 输出 {config: "wantu"}{% endhighlight %}

liftoff 主要解决了几件事情

  1. 定义、查找并读取配置文件
  2. 作为命令处理的入口

liftoff 的用法也较为简单

{% highlight js %}const Liftoff = require("liftoff"); const APP_NAME = "rocketz"; const app = new Liftoff({ name: APP_NAME, configName: `.${APP_NAME}`, extensions: { "rc": null } }); // 启动应用 app.launch({ cwd: process.cwd(), configPath: __dirname }, function( env ) { // 命令和 flag 的处理逻辑 }); {% endhighlight %}

inquirer 可以让你设置一些「问答题」

通过在命令行中的一步步操作

完成对配置文件内容的设置

meow＋liftoff＋inquirer＝一个完整的命令行工具

服务端开发

这是另一种较高级的 node 应用

也是给前端带来新希望的一种应用

做服务端开发离不开 web 框架

在我眼里总共有两种类型

  1. 全栈式
  2. REST

所谓的「全栈式」框架

就是完整地提供了 MVC 支持的框架

轻量点的有 Express、Koa

它们都太轻量

以至于需要自己去东拼西凑成一套完整的框架

不太适合快速开发

有个叫做 Sails 的框架

看名字也许会想到了什么

没错，就是那辆小火车

这是一个借鉴 Ruby on Rails 的大而全的框架

挺适合快速开发的

然而，在现在这种多终端的使用环境下

全栈式的框架显得不那么灵活

服务端只需要通过 API 接口提供数据就好了

restify 就是为此而生

Node.js 改变了我

让我从纯前端渐渐转变为可以贯穿 web 栈的伪·全栈

我的废话说完了
